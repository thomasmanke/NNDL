Image Segmentation: Introduction & Motivation

What is Image Segmentation?

Image segmentation is the process of partitioning a digital image into multiple segments (sets of pixels). Unlike classification which assigns a single label to an entire image, segmentation assigns a label to each pixel, resulting in a pixel-wise classification map (mask).

Classification vs. Segmentation
  • Classification: Image -> Single label (e.g., “cat”, “dog”, “tumor”)
  • Segmentation: Image -> Pixel-wise labels (e.g., which pixels belong to the tumor?)

Segmentation is a dense prediction task where the output has the same spatial dimensions as the input.

Medical Imaging Applications

Image segmentation is particularly crucial in medical imaging for:

  • Tumor detection and measurement: Identifying cancerous regions in MRI/CT scans
  • Organ segmentation: Delineating organs for surgical planning
  • Lesion quantification: Measuring disease progression
  • Treatment planning: Targeting radiation therapy precisely
Segmentation Challenges
  • segmenation crucial for diagnoses and treatment plan
  • multiple MRI modalities
  • individual variability, heterogeneity and complexity
  • technical artefacts
  • time-consuming and subjective
  • BraTS Challenge (2012-)

Historical Context

  • 1970s-1980s: Early thresholding and region-growing methods
  • 1990s: Active contour models (Snakes), watershed algorithms
  • 2000s: Graph-cut methods, conditional random fields
  • 2015: U-Net revolution - Ronneberger, Fischer, Brox introduced the U-Net architecture for biomedical image segmentation, winning the ISBI cell tracking challenge
  • 2017+: Deep learning dominates with variants like U-Net++, DeepLab, Mask R-CNN

Reference: U-Net: Convolutional Networks for Biomedical Image Segmentation

U-Net Architecture: The Encoder-Decoder Paradigm

Why U-Net?

U-Net is specifically designed for biomedical image segmentation and addresses key challenges:

  1. Limited training data in medical imaging
  2. Precise localization requirements (pixel-level accuracy)
  3. Context understanding - needs both high-resolution features and semantic context

Architecture Overview

The U-Net architecture has a symmetric encoder-decoder structure with skip connections:

Key Components
Encoder (Contracting Path)
  • Purpose: Capture context and high-level features
  • Operations: 3x3 convolution, ReLU, 2x2 max pooling
  • Channel progression: 64 -> 128 -> 256 -> 512 -> 1024
Bottleneck
  • Connects encoder and decoder
  • Highest channel count, smallest spatial dimensions
Decoder (Expansive Path)
  • Purpose: Enable precise localization using upsampled features
  • Operations: 2x2 up-convolution, 3x3 convolution, ReLU
  • Channel progression: 1024 -> 512 -> 256 -> 128 -> 64
Skip Connections
  • Concatenate Feature maps (encoder) with upsampled maps (decoder)

Why Skip Connections Matter

Without skip connections, the decoder struggles to recover fine details. They provide: - High-resolution features from earlier layers - Boundary information lost in deep layers - Context for precise edge delineation

This is especially important in medical imaging where boundary accuracy is critical.

Loss Function: Dice Loss

For segmentation, Dice Loss is more appropriate than Cross-Entropy:

Dice(A, B) = (2 * |A intersect B|) / (|A| + |B|)

Dice Loss = 1 - Dice(A, B)

Code 
loss_fn = DiceLoss(sigmoid=True, reduction="mean")
dice_metric = DiceMetric(reduction="mean", num_classes=NUM_CLASSES)
iou_metric = MeanIoU(reduction="mean")
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

Why Dice Loss?

  • Handles class imbalance: Medical images often have small regions of interest
  • Smooth gradients: Better for optimization
  • Interpretability: Dice coefficient directly measures overlap quality (0-1 scale)

Training Loop for Segmentation

Code 
NUM_EPOCHS = 20

history = {
    'train_loss': [], 'val_loss': [],
    'train_dice': [], 'val_dice': [],
    'train_iou': [], 'val_iou': [],
}

for epoch in range(NUM_EPOCHS):
    model.train()
    train_loss = 0
    
    for img_batch, mask_batch in train_loader:
        img_batch = img_batch.to(device)
        mask_batch = mask_batch.to(device)
        
        optimizer.zero_grad()
        outputs = model(img_batch)
        loss = loss_fn(outputs, mask_batch)
        loss.backward()
        optimizer.step()
        train_loss += loss.item()
        
        dice_metric(torch.sigmoid(outputs), mask_batch)
        iou_metric(torch.sigmoid(outputs), mask_batch)
    
    avg_train_loss = train_loss / len(train_loader)
    avg_train_dice = dice_metric.aggregate().item()
    avg_train_iou = iou_metric.aggregate().item()
    
    model.eval()
    val_loss = 0
    with torch.no_grad():
        for img_batch, mask_batch in val_loader:
            img_batch = img_batch.to(device)
            mask_batch = mask_batch.to(device)
            outputs = model(img_batch)
            loss = loss_fn(outputs, mask_batch)
            val_loss += loss.item()
            dice_metric(torch.sigmoid(outputs), mask_batch)
            iou_metric(torch.sigmoid(outputs), mask_batch)
    
    avg_val_loss = val_loss / len(val_loader)
    avg_val_dice = dice_metric.aggregate().item()
    avg_val_iou = iou_metric.aggregate().item()
    
    history['train_loss'].append(avg_train_loss)
    history['val_loss'].append(avg_val_loss)
    history['train_dice'].append(avg_train_dice)
    history['val_dice'].append(avg_val_dice)
    history['train_iou'].append(avg_train_iou)
    history['val_iou'].append(avg_val_iou)
    
    dice_metric.reset()
    iou_metric.reset()
    
    print(f"Epoch {epoch+1}/{NUM_EPOCHS}: Loss: {avg_train_loss:.4f}/{avg_val_loss:.4f}, "

Comparison with Other Architectures

U-Net vs. Fully Convolutional Network (FCN)

Feature U-Net FCN
Architecture Encoder-Decoder with skip connections Encoder only (modified classifier)
Upsampling Transposed convolutions + skip connections Transposed convolutions only
Resolution Full resolution output Lower resolution
Training Data Efficient with limited data Requires large dataset
Accuracy High (especially for small objects) Moderate
Use Case Medical imaging, precise boundaries General segmentation

U-Net vs. DeepLab

Feature U-Net DeepLab
Atrous Convolutions No Yes (multi-scale context)
Skip Connections Yes No
ASPP Module No Yes (Atrous Spatial Pyramid Pooling)
Complexity Moderate High
Use Case Medical imaging Scene parsing, autonomous driving

When to Use U-Net?

Use U-Net when: - Limited training data (common in medical imaging) - Precise boundary delineation needed - Objects vary in size - Real-time or near real-time inference

Consider alternatives when: - Abundant labeled data available - Objects are very large relative to image size - Instance segmentation needed (not semantic segmentation) - Working with 3D volumes (use 3D U-Net instead)

Automated segmentation transforms medical imaging workflows: - Radiology: Automated tumor volume measurement in MRI/CT - Pathology: Cell segmentation in digital pathology slides - Cardiology: Left ventricle segmentation in cardiac MRI - Neurology: Hippocampus segmentation for Alzheimer’s diagnosis - Oncology: Radiation therapy planning

Challenges in Medical Image Segmentation

Key Challenges
  • Class Imbalance: Tumors occupy <1% of image pixels
  • Data Scarcity: Annotating requires expert radiologists
  • Domain Shift: Models trained on one scanner/hospital may not generalize
  • 3D Data: Many medical images are volumetric (CT, MRI)
  • Multi-modal: Combining different imaging modalities
  • Ethical Considerations: Model errors can have serious consequences

## Image Segmentation: Introduction & Motivation

### What is Image Segmentation?

Image segmentation is the process of partitioning a digital image into multiple segments (sets of pixels). Unlike classification which assigns a single label to an entire image, segmentation assigns a label to each pixel, resulting in a pixel-wise classification map (mask).

::: {.callout-note}
### Classification vs. Segmentation
- Classification: Image -> Single label (e.g., "cat", "dog", "tumor")
- Segmentation: Image -> Pixel-wise labels (e.g., which pixels belong to the tumor?)

Segmentation is a dense prediction task where the output has the same spatial dimensions as the input.
:::

### Medical Imaging Applications


Image segmentation is particularly crucial in medical imaging for:

- Tumor detection and measurement: Identifying cancerous regions in MRI/CT scans
- Organ segmentation: Delineating organs for surgical planning
- Lesion quantification: Measuring disease progression
- Treatment planning: Targeting radiation therapy precisely

![Brain tumor segmentation in MRI. Bonato et al. (2026)](images/brats_challenge.jpg)


::: {.callout-note}
### Segmentation Challenges
- segmenation crucial for diagnoses and treatment plan
- multiple MRI modalities
- individual variability, heterogeneity and complexity
- technical artefacts
- time-consuming and subjective
- [BraTS Challenge (2012-)](https://www.synapse.org/Synapse:syn53708126/wiki/626320){target="_blank" rel="noopener"}
:::



### Historical Context

- 1970s-1980s: Early thresholding and region-growing methods
- 1990s: Active contour models (Snakes), watershed algorithms
- 2000s: Graph-cut methods, conditional random fields
- 2015: U-Net revolution - Ronneberger, Fischer, Brox introduced the U-Net architecture for biomedical image segmentation, winning the ISBI cell tracking challenge
- 2017+: Deep learning dominates with variants like U-Net++, DeepLab, Mask R-CNN

## U-Net Architecture: The Encoder-Decoder Paradigm

### Why U-Net?

U-Net is specifically designed for biomedical image segmentation and addresses key challenges:

1. Limited training data in medical imaging
2. Precise localization requirements (pixel-level accuracy)
3. Context understanding - needs both high-resolution features and semantic context

### Architecture Overview

The U-Net architecture has a symmetric encoder-decoder structure with skip connections:

![U-Net architecture. Ronneberger, Fischher & Brox (2015)](images/u-net-architecture.png)


::: {.callout-important}
#### Key Components

##### Encoder (Contracting Path)
- Purpose: Capture context and high-level features
- Operations: 3x3 convolution, ReLU, 2x2 max pooling
- Channel progression: 64 -> 128 -> 256 -> 512 -> 1024

##### Bottleneck
- Connects encoder and decoder
- Highest channel count, smallest spatial dimensions

##### Decoder (Expansive Path)
- Purpose: Enable precise localization using upsampled features
- Operations: 2x2 up-convolution, 3x3 convolution, ReLU
- Channel progression: 1024 -> 512 -> 256 -> 128 -> 64

##### Skip Connections
- Concatenate Feature maps (encoder) with upsampled maps (decoder)

:::

### Why Skip Connections Matter

Without skip connections, the decoder struggles to recover fine details. They provide:
- High-resolution features from earlier layers
- Boundary information lost in deep layers
- Context for precise edge delineation

This is especially important in medical imaging where boundary accuracy is critical.

### Loss Function: Dice Loss

For segmentation, Dice Loss is more appropriate than Cross-Entropy:

Dice(A, B) = (2 * |A intersect B|) / (|A| + |B|)

Dice Loss = 1 - Dice(A, B)

```{python}
#| label: define_loss_and_metrics

loss_fn = DiceLoss(sigmoid=True, reduction="mean")
dice_metric = DiceMetric(reduction="mean", num_classes=NUM_CLASSES)
iou_metric = MeanIoU(reduction="mean")
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
```

### Why Dice Loss?
- Handles class imbalance: Medical images often have small regions of interest
- Smooth gradients: Better for optimization
- Interpretability: Dice coefficient directly measures overlap quality (0-1 scale)

## Training Loop for Segmentation

```{python}
#| label: train_segmentation
#| code-fold: true

NUM_EPOCHS = 20

history = {
    'train_loss': [], 'val_loss': [],
    'train_dice': [], 'val_dice': [],
    'train_iou': [], 'val_iou': [],
}

for epoch in range(NUM_EPOCHS):
    model.train()
    train_loss = 0
    
    for img_batch, mask_batch in train_loader:
        img_batch = img_batch.to(device)
        mask_batch = mask_batch.to(device)
        
        optimizer.zero_grad()
        outputs = model(img_batch)
        loss = loss_fn(outputs, mask_batch)
        loss.backward()
        optimizer.step()
        train_loss += loss.item()
        
        dice_metric(torch.sigmoid(outputs), mask_batch)
        iou_metric(torch.sigmoid(outputs), mask_batch)
    
    avg_train_loss = train_loss / len(train_loader)
    avg_train_dice = dice_metric.aggregate().item()
    avg_train_iou = iou_metric.aggregate().item()
    
    model.eval()
    val_loss = 0
    with torch.no_grad():
        for img_batch, mask_batch in val_loader:
            img_batch = img_batch.to(device)
            mask_batch = mask_batch.to(device)
            outputs = model(img_batch)
            loss = loss_fn(outputs, mask_batch)
            val_loss += loss.item()
            dice_metric(torch.sigmoid(outputs), mask_batch)
            iou_metric(torch.sigmoid(outputs), mask_batch)
    
    avg_val_loss = val_loss / len(val_loader)
    avg_val_dice = dice_metric.aggregate().item()
    avg_val_iou = iou_metric.aggregate().item()
    
    history['train_loss'].append(avg_train_loss)
    history['val_loss'].append(avg_val_loss)
    history['train_dice'].append(avg_train_dice)
    history['val_dice'].append(avg_val_dice)
    history['train_iou'].append(avg_train_iou)
    history['val_iou'].append(avg_val_iou)
    
    dice_metric.reset()
    iou_metric.reset()
    
    print(f"Epoch {epoch+1}/{NUM_EPOCHS}: Loss: {avg_train_loss:.4f}/{avg_val_loss:.4f}, "
          f"Dice: {avg_train_dice:.4f}/{avg_val_dice:.4f}, IoU: {avg_train_iou:.4f}/{avg_val_iou:.4f}")
```

## Comparison with Other Architectures

### U-Net vs. Fully Convolutional Network (FCN)

| Feature | U-Net | FCN |
|---------|-------|-----|
| Architecture | Encoder-Decoder with skip connections | Encoder only (modified classifier) |
| Upsampling | Transposed convolutions + skip connections | Transposed convolutions only |
| Resolution | Full resolution output | Lower resolution |
| Training Data | Efficient with limited data | Requires large dataset |
| Accuracy | High (especially for small objects) | Moderate |
| Use Case | Medical imaging, precise boundaries | General segmentation |

### U-Net vs. DeepLab

| Feature | U-Net | DeepLab |
|---------|-------|--------|
| Atrous Convolutions | No | Yes (multi-scale context) |
| Skip Connections | Yes | No |
| ASPP Module | No | Yes (Atrous Spatial Pyramid Pooling) |
| Complexity | Moderate | High |
| Use Case | Medical imaging | Scene parsing, autonomous driving |

### When to Use U-Net?

Use U-Net when:
- Limited training data (common in medical imaging)
- Precise boundary delineation needed
- Objects vary in size
- Real-time or near real-time inference

Consider alternatives when:
- Abundant labeled data available
- Objects are very large relative to image size
- Instance segmentation needed (not semantic segmentation)
- Working with 3D volumes (use 3D U-Net instead)
