Review

  • neural networks: layers of neurons connected by weight
  • in each neuron: weights + bias + non-lenar activation function
  • optimization: gradient descent, SGD , momentum, ADAM
  • backprogation: gradients with chain rule on computational graph
  • technical: vanishing gradients â€“> batch norm, weight initialization
  • overfitting and generalization: data augmentation, regularization (L2, dropout)

Why not add more layers

  • NN with one hidden layer are universal function approximators
  • otpimization hard: finding parameters to give optimal decision boundary (for classification)
  • no structure; just brute force
  • black boxes: parameter interpretation
  • performance palteau

â€“> more structure: CNN, RNN, Transformers â€“> from classifiers to generators

Problem:

fully connected layers: input(1000 x 1000 x 3 pixels) â€“> 1 hidden layer(1000 neurons) â€“> 3 billion parameters

restrict capacity of each layer â€“> weight sharing

invariance assumption: should not matter where feature is

CNN: start point for subsequence task - classification - object localization (bounding box) - object detection (multiple bounding boxes) - instance segmentation (point-wise classification)

Python Implementation

Define Model:

Code 
import torch
import torch.nn as nn
import torchvision.models as models
import numpy as np

class my_model(nn.Module):
    """ 
    My simple Convolutional Network
    input shape: [,3,224,224] 
    output shape: [,10]
    """
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1)
        self.pool = nn.MaxPool2d(kernel_size=2, stride=2)
        self.conv2 = nn.Conv2d(64, 64, kernel_size=3, stride=1, padding=1)
        self.flatten = nn.Flatten()
        self.fc = nn.Linear(112 * 112 * 64, 10)

    def forward(self, x):
        x = self.conv1(x)
        x = self.pool(x)
        x = self.conv2(x)
        x = self.flatten(x) 
        x = self.fc(x)
        return x

Notice that each model has a defined input shape (e.g. [,3,224,224]) and a defined output (e.g [,10]). The inner layers should work such that the dimensions match.

Below I show how to use, hooks that allow for efficient manipulation of the forward path, such that all quantities of interest can be tracked. Here I will track the shapes

Define Hook

Code 
def describe_model_forward(model, input_tensor):
    """ 
    describe_model_forward collects information on the shapes of parameters and outputs 
    on each computational layer of a neural network
    """
    layer_info = []
    hooks = []

    # define a "hook" function that can be passed to the model
    # and evaluate as each layer is run in the foward path
    def register_hook(module):
        def hook(module, input, output):
            name = module.__class__.__name__
            # assumption: output of model is single tensor
            output_shape = tuple(output.shape)
            param_shapes = [tuple(p.shape) for p in module.parameters() if p.requires_grad]

            layer_info.append((name, output_shape, param_shapes))

        # add hook to computational layers (not containers Sequential, ModuleList)
        # Warning: layer and module is ALMOST synonymous, but modules may also be container of layers
        # here we exclude them conditionally
        # This was trial and error. I'm not sure if this exlcusion list is exhaustive in general
        if not isinstance(module, nn.Sequential) and not isinstance(module, nn.ModuleList):
            hooks.append(module.register_forward_hook(hook))

    # Recursively add register_hook() function to each computational layer
    model.apply(register_hook)

    # Run forward pass
    with torch.no_grad():
        _ = model(input_tensor)

    # Remove hooks to clean memory
    for h in hooks:
        h.remove()

    return layer_info

Habits and Recommendations

  • use input size \(LxL = 2^n\) (e.g. 512)
  • use stride \(S=1\)
  • use padding \(P= (F-1)/2\) to retain input size â€“> multiple CONV layers
  • pooling: \(F=2 S=2\) (size reduction: \(LxL --> L/2xL/2\) !!! - aggressive reduction,

```{python}
# Print out information
for (name, out_shape, param_shape) in layer_info:
    # assume that there is always an output shape
    n_out = int(np.prod(out_shape))
    n_params = 0
    # parameters shape maybe empty (e.g ReLU --> 0 params)
    if len(param_shape) == 2:
        n_weights = np.prod(param_shape[0])
        n_bias = np.prod(param_shape[1])
        n_params = int(n_weights + n_bias)

    print(f"Layer {name:<14} \
        output: {str(out_shape):<25} {n_out:<10,}\
        Parameters: {str(param_shape):<15} {n_params:,}"
    )
```



### Habits and Recommendations

- use input size $LxL = 2^n$ (e.g. 512)
- use stride $S=1$
- use padding $P= (F-1)/2$ to retain input size --> multiple CONV layers
- pooling: $F=2 S=2$ (size reduction: $LxL --> L/2xL/2$ !!! - aggressive reduction,
