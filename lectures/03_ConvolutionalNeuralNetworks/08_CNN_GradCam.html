Dealing with ReLU

Notice that the ReLU of the VGG16 model are shown as ReLU(inplace=True) In other words; ReLU(x) overwrites the orginal tensor \(x\).

This saves memory, but later we will need access to the original activations (and gradients). Therefore we set inplace=False.

Code 
for m in model.modules():
    if isinstance(m, torch.nn.ReLU):
        m.inplace = False

# check that RelU has no inplace=TRUE anylonger

def get_weights(model, layer_idx):
    weights = model.features[layer_idx].weight
    # normalize
    f_min = weights.min()
    f_max = weights.max()
    weights = (weights - f_min) / (f_max - f_min + 1e-8)

    return weights

Optimizing Input: Activation Maximization

For each layer \(l\) and channel \(c\) we can define the mean activation (single scalar) for a given input image \(x\) \[ J_{l,c}(x) = \frac{1}{HW} \sum_{ij} f_{l,c}(x) \]

Idea

optimize input image \(x\) to maximize \(J_{l,c}(x)\)

\[ \underset{x}{\mbox{argmax}} \left( J_{l,c}(x) - \lambda R(x) \right) \]

Notes
  • c.f.loss minimization \(L(w)\): optimize \(w\)!
  • network parameters \(w\) fixed
  • gradient ascent: iterative optimization starting from random image
  • dependence on definition of \(J_{l,c}\) (other definitions possible)
  • dependence on regularization \(R(x)\) (smoothness, priors etc)
  • independent of data set !
Implementations
  • Lucid (TF 2018) distill.pub (Olah et al.)
  • Lucent (PyTorch 2020) (L. Kiat & T. Lieberum)
Uses
  • detect dead filters
  • compare models

Interprete Predictions

Given a class prediction \(x \to c\), how would changes in one layer effect the output value.

Saliency

Code 
# track gradients with respect to x
x = x.clone().detach().requires_grad_(True)

model.eval()       # set model into evaluation mode
model.zero_grad()  # zero all gradients

logits = model(x)                          # logits [1,1000]
c_id = logits.argmax(dim=1).item()         # pred. class
print('class: ', c_id, class_labels[c_id])

score = logits[0,c_id]     # class prediction score
score.backward()           # backward prop to calculate

saliency = x.grad.detach().abs()                  # [1,3,H,W]
saliency = saliency.max(dim=1, keepdim=True)[0]   # [1,1,H,W] max over c
saliency = saliency.detach()[0,0].cpu().numpy()   # [H,W]
# Normalize to [0,1]
saliency = (saliency - saliency.min()) / (saliency.max() - saliency.min() + 1e-8)

# Keep original image for overlay [H,W,C]
# resize to match x and normalize to [0,1]
img_for_overlay = img.resize(x.shape[2:])
img_for_overlay = np.float32(img_for_overlay) / 255.0

plt.figure(figsize=(8,4))
plt.subplot(1,2,1); plt.imshow(img_for_overlay) 
plt.axis('off'); plt.title('Original')
plt.subplot(1,2,2); plt.imshow(saliency, cmap='gray')
plt.axis('off'); plt.title('Saliency')

Grad-CAM

Principle Idea

Importance of a pixel \((i,j)\) is related to

  • size of activation scores in last layer \(A_{ij}^k\)
  • importance of changes in score for class prediction \(c\): gradient \(\delta y^c / \delta A_{ij}^k\)
  • weighted average over all channel contributions from last conv layer

\[ L_{ij} = ReLU\left(\sum_k w_k A_{ij}^k\right) \]

Some heuristics:

  • consider only positive contributions (\(ReLU\))
  • channel weight \(w_k\) (spatial average of gradients)

\[ w_k = \frac{1}{Z} \sum_{i,j} \frac{\partial y^c}{\partial A_{ij}^k} = \frac{1}{Z} \sum_{i,j} g_{ij}^k~~~(Z=H'W') \]

(usually we focus on a single class: drop \(c\) for simplcity)

Activations \(A_{ij}^k\) and gradients \(g_{ij}^k\) from forward and backpropagation.

More heuristics: Grad-Cam++

Estimates for \(w_k\) can be improved

\[ w_k = \sum_{i,j} \alpha_{ij}^k ReLU(g_{ij}^k) \]

with pixel specific weights

\[ \alpha_{ij}^k = \frac{(g_{ij}^k)^2}{ 2(g_{ij}^k)^2 + A^k (g_{ij}^k)^3 + \epsilon} ~~~~~A^k = \sum_{a,b} A_{ab}^k \]

The goals with this are:

  • pixels with large positive gradient should contribute more to the weight.
  • globally active channels should be downgraded because we want localization: discourage large \(A^k\) in denominator
  • keep only positive evidence: \(ReLU(g_{ij})\)

### Dealing with ReLU

Notice that the ReLU of the VGG16 model are shown as `ReLU(inplace=True)`
In other words; `ReLU(x)` overwrites the orginal tensor $x$.

This saves memory, but later we will need access to the original activations (and gradients). Therefore we set `inplace=False`.

for m in model.modules():

## Optimizing Input: Activation Maximization

For each layer $l$ and channel $c$ we can define the mean activation (single scalar) for a given input image $x$
$$
J_{l,c}(x) = \frac{1}{HW} \sum_{ij} f_{l,c}(x)
$$

::: {.callout-note}
#### Idea
optimize input image $x$ to maximize $J_{l,c}(x)$
:::

$$
\underset{x}{\mbox{argmax}} \left( J_{l,c}(x) - \lambda R(x) \right)
$$

::: {.callout-note}
#### Notes
- c.f.loss minimization $L(w)$: optimize $w$! 
- network parameters $w$ fixed
- **gradient ascent**: iterative optimization starting from random image
- dependence on definition of $J_{l,c}$ (other definitions possible)
- dependence on regularization $R(x)$ (smoothness, priors etc)
- independent of data set !
:::



::: {.callout-note}
#### Implementations
- Lucid (TF 2018) [distill.pub (Olah et al.)](https://distill.pub/2018/building-blocks){target="_blank" rel="noopener"}
- Lucent (PyTorch 2020) [(L. Kiat & T. Lieberum)](https://lucent.readthedocs.io/en/latest/index.html){target="_blank" rel="noopener"}
:::

### Saliency

```{python}
#| label: saliency

# track gradients with respect to x
x = x.clone().detach().requires_grad_(True)

model.eval()       # set model into evaluation mode
model.zero_grad()  # zero all gradients

logits = model(x)                          # logits [1,1000]
c_id = logits.argmax(dim=1).item()         # pred. class
print('class: ', c_id, class_labels[c_id])

score = logits[0,c_id]     # class prediction score
score.backward()           # backward prop to calculate

saliency = x.grad.detach().abs()                  # [1,3,H,W]
saliency = saliency.max(dim=1, keepdim=True)[0]   # [1,1,H,W] max over c
saliency = saliency.detach()[0,0].cpu().numpy()   # [H,W]
# Normalize to [0,1]
saliency = (saliency - saliency.min()) / (saliency.max() - saliency.min() + 1e-8)

# Keep original image for overlay [H,W,C]
# resize to match x and normalize to [0,1]
img_for_overlay = img.resize(x.shape[2:])
img_for_overlay = np.float32(img_for_overlay) / 255.0

plt.figure(figsize=(8,4))
plt.subplot(1,2,1); plt.imshow(img_for_overlay) 
plt.axis('off'); plt.title('Original')
plt.subplot(1,2,2); plt.imshow(saliency, cmap='gray')
plt.axis('off'); plt.title('Saliency')
plt.tight_layout(); plt.show()
```


### Grad-CAM

::: {.callout-note}
#### Principle Idea
Importance of a pixel $(i,j)$ is related to  

- size of activation scores in **last layer** $A_{ij}^k$
- importance of changes in score for class prediction $c$: **gradient** $\delta y^c / \delta A_{ij}^k$
- **weighted average** over all channel contributions from last conv layer
:::

$$
L_{ij} = ReLU\left(\sum_k w_k A_{ij}^k\right)
$$

Some heuristics:

- consider only positive contributions ($ReLU$)
- channel weight $w_k$ (spatial average of gradients)

$$
w_k = \frac{1}{Z} \sum_{i,j} \frac{\partial y^c}{\partial A_{ij}^k} = \frac{1}{Z} \sum_{i,j} g_{ij}^k~~~(Z=H'W')
$$

(usually we focus on a single class: drop $c$ for simplcity)


Activations $A_{ij}^k$ and gradients $g_{ij}^k$ from forward and backpropagation.

![Graphical Summary. GradCam vs. Saliency.](images/interpretation.jpg)

::: {.callout-note collapse="true"}
#### More heuristics: `Grad-Cam++`
Estimates for $w_k$ can be improved

$$
w_k = \sum_{i,j} \alpha_{ij}^k ReLU(g_{ij}^k)
$$

with pixel specific weights

$$
\alpha_{ij}^k = \frac{(g_{ij}^k)^2}{ 2(g_{ij}^k)^2 + A^k (g_{ij}^k)^3 + \epsilon} ~~~~~A^k = \sum_{a,b} A_{ab}^k
$$

The goals with this are: 

- pixels with large positive gradient should contribute more to the weight. 
- globally active channels should be downgraded because we want localization: discourage large $A^k$ in denominator
- keep only positive evidence: $ReLU(g_{ij})$
:::


```{python}
#| label: grad_cam_pp
#| eval: false
#| echo: false

import torch
import torch.nn.functional as F
import numpy as np
import matplotlib.pyplot as plt

def gradcam_pp(model, x, target_layer, class_idx, up_size=None, eps=1e-8):
    """
    model: nn.Module in eval() mode
    x:     input tensor of image [N,3,H,W]
    target_layer: a conv layer to explain (e.g., model.features[28] for VGG16)
    class_idx: int (target class id)
    up_size: (H, W) to upsample CAM to; defaults to input spatial size
    returns: CAM [N,H,W] normalized to [0,1]

    Refs: 
    - https://medium.com/@stepanulyanin/implementing-grad-cam-in-pytorch-ea0937c31e82
    - https://medium.com/@kumar_devesh/visualization-and-model-explanations-in-convolutional-neural-networks-70cdfa4c57ec
    """

    acts = []
    grads = []

    # define hooks to get activations (forward) and gradients (backward)
    def fwd_hook(_, __, out):
        acts.append(out.detach())             # [N,C,H',W']

    def bwd_hook(_, grad_in, grad_out):
        grads.append(grad_out[0].detach())    # [N,C,H',W']

    # register hooks in target layer
    h1 = target_layer.register_forward_hook(fwd_hook)
    h2 = target_layer.register_full_backward_hook(bwd_hook)

    # set grads to zero
    model.zero_grad()
    logits = model(x)                 # [N,num_classes]
    # score for class (usually the to prediction)
    score = logits[:, class_idx].sum()   
    score.backward()
    
    h1.remove(); h2.remove()         # remove hooks

    A = acts[0]                      # activations [N,C,H',W']
    G = grads[0]                     # gradients:  [N,C,H',W']

    # Grad-CAM weights (not used)
    # Z = G.shape[2]*G.shape[3]
    # weights = G.sum(dim=(2,3), keepdim=True) / Z  # [N,C,1,1]

    # Grad-CAM++ weights
    G2 = G * G                            # g^2
    G3 = G2 * G                           # g^3
    sumA = A.sum(dim=(2,3), keepdim=True) # \sum_{a,b} A_{ab}^k  [1,C,1 ,1]

    alpha = G2 / (2 * G2 + sumA * G3 + eps)   # [N,C,H',W']
    weights = (alpha * F.relu(G)).sum(dim=(2,3), keepdim=True)  # [N,C,1,1]

    # Weighted sum over channels + ReLU
    cam = F.relu((weights * A).sum(dim=1, keepdim=True))        # [N,1,H',W']

    # Normalize per-sample to [0,1]
    N = cam.shape[0]
    cam = cam.view(N, -1)                                       # [N,H'*W']
    cam = (cam - cam.min(dim=1, keepdim=True).values) / (cam.max(dim=1, keepdim=True).values - cam.min(dim=1, keepdim=True).values + eps)                                # [N,H'*W'] in [0,1]
    cam = cam.view(-1, 1, *acts[0].shape[-2:])                 # [N,1,H',W']

    # up_size (per default to original image size)
    if up_size is None:
        up_size = x.shape[-2:]  # input H,W
    cam_up = F.interpolate(cam, size=up_size, mode='bilinear', align_corners=False)  # [N,1,H,W]
    return cam_up[:,0]  # [N,H,W]


print('c_id: ', c_id)

target_size = x.shape[-2:] # can be adjusted
cam_pp = gradcam_pp(model, x, target_layer=model.features[28], class_idx=c_id, up_size=target_size)  # [1,H,W]
cam_pp = cam_pp[0].cpu().numpy()   # [H,W], in [0,1]

img_np = np.float32(img.resize((cam_pp.shape[1], cam_pp.shape[0]))) / 255.0
heatmap = plt.cm.jet(cam_pp)[..., :3]
overlay = 0.5 * img_np + 0.5 * heatmap
overlay = np.clip(overlay, 0, 1)

plt.figure(figsize=(8,4))
plt.subplot(1,2,1); plt.imshow(img_np); plt.axis('off'); plt.title(f'Original ')
plt.subplot(1,2,2); plt.imshow(overlay); plt.axis('off'); plt.title(f'Grad-CAM++ (class {c_id})')
plt.tight_layout(); plt.show()
```
