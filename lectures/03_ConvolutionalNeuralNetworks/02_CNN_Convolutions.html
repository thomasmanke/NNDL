1D Conv

Convolutions combine two functions by sliding one (the kernel \(g(x)\)) over the other

An image:

A formula: \[ (f \ast g)(x) = \sum_{k=-\infty}^{\infty} f(k) \cdot g(x-k) \]

sum is over all \(k\), but in practice \(g(x-k) = 0\) when \(|x-k| \gg 0\)

This can be used conveniently for

  • smoothing/averaging
  • change detection (derivatives)
  • pattern detection
  • sharpening
Code 
import numpy as np
import matplotlib.pyplot as plt
from scipy.signal import convolve

# Create a signal with edges
x = np.linspace(-10, 10, 400)
f = np.sin(x) + 0.1*np.random.randn(len(x))  # noisy sinus

# Kernels
xk = np.linspace(-3, 3, 51)
sigma = 1
g1 = np.exp(-(xk**2)/(2*sigma**2)); 
g1 = g1 / g1.sum()             # Gaussian for smoothing/averaging
g2 = np.array([-1, 1])         # 1st derivative
g3 = np.array([1, -2, 1])      # 2nd derivative (Laplacian)
g4 = np.array([-1, 2, -1])     # High-pass filter (sharpening)

# Convolve
fs = convolve(f, g1, mode='same') # smoothed signal
f1 = convolve(fs, g2, mode='same') # 1st derivative
f1s = convolve(f1, g1, mode='same') # smoothed 1st derivativ
f2 = convolve(f1s, g3, mode='same') # 2nd derivative

# remove boundary effects
pad = 81
x = x[pad:-pad]
f = f[pad:-pad]
fs = fs[pad:-pad]
f1 = f1[pad:-pad]
f2 = f2[pad:-pad]

# Plot: 
# kernels are scaled arbitrarly for visibility
fsize = 12
plt.figure(figsize=(7, 7))
plt.subplot(4,1,1)
plt.plot(x, f)
plt.title("Original Data", fontsize=fsize)

plt.subplot(4,1,2)
plt.plot(x, fs)
plt.bar(xk-1, g1*20, color="orange", width=0.05)
plt.title("Smoothing/Averaging: fs = f * G  \n Gaussian Kernel $G \\propto \\exp{(-x^2)}$", fontsize=fsize)

plt.subplot(4,1,3)
plt.plot(x, f1); 
plt.bar(x[[99,100]], g2/20, color="orange",width=0.05)
plt.title("Change Detection: f1 = fs * D \n 1st derivative D = [-1, +1]", fontsize=fsize)

plt.subplot(4,1,4)
plt.plot(x, f2); 
plt.bar(x[[99,100,101]], g3/10000, color="orange",width=0.05)
plt.title("Curvature Detection: f2 = (f1 * G) * L \n 2nd derivative L = [1, -2, 1]", fontsize=fsize)

plt.tight_layout()
plt.show()
Note
  • appropriate convolutions can extract features
  • many different kernels possible
  • find different (more suitable) data representations
  • kernel rules are shift-invariant and independent of data size
  • controlled by few parameters

2D Conv

Illustration

Real Examples

Below I use 2 filters for vertical and horizontal edge detection. \[ K_1 = \begin{bmatrix} -1 & 0 & 1 \\ -1 & 0 & 1 \\ -1 & 0 & 1 \end{bmatrix} \qquad K_2 = \begin{bmatrix} -1 & -1 & -1 \\ 0 & 0 & 0 \\ 1 & 1 & 1 \end{bmatrix} \]

Code 
from torchvision import datasets, transforms
from scipy.signal import convolve2d

transform = transforms.ToTensor()
dataset = datasets.CIFAR10(root="./data", train=False, download=True, transform=transform)

img, label = dataset[24]              # RGB img: [3,32,32]
img_gray = img.mean(dim=0).numpy()   # grey img: [32, 32]

K1 = np.array([
    [-1,  0,  1],
    [-1,  0,  1],
    [-1,  0,  1]
], dtype=float)
K2 = np.array([
    [-1,  -1,  -1],
    [0,  0,  0],
    [1,  1,  1]
], dtype=float)

img1 = convolve2d(img_gray, K1, mode="same", boundary="symm")
img2 = convolve2d(img_gray, K2, mode="same", boundary="symm")

img1 = (img1 - img1.min()) / (img1.max() - img1.min())
img2 = (img2 - img2.min()) / (img2.max() - img2.min())

plt.figure(figsize=(9,3))

plt.subplot(1, 3, 1)
plt.imshow(img_gray, cmap='gray')
plt.title("original CIFAR-10 image (gray)")
plt.axis("off")

plt.subplot(1, 3, 2)
plt.imshow(img1, cmap='gray')
plt.title("vertical edge filter K1")
plt.axis("off")

plt.subplot(1, 3, 3)
plt.imshow(img2, cmap='gray')
plt.title("horizontal edge filter K2")
plt.axis("off")

plt.show()
Note
  • Each filter creates a new “feature map”
  • CNN will learn filter weights
  • many different filters and corresponding feature maps
  • many layers of filters
  • emergence of “objects” and “concepts” in deep layers

Generalization & New Jargon

  • input channels: RGB images [C=3, H, W]

  • kernels=filters=neurons: [K,C,3,3]

    • \(3 \times 3\) filter: C input channels \(\to\) K output channels (depth)
    • K different filters for different patterns (e.g. horizontal, vertical lines)
    • filters have same channel depth as input
    • filter parameters are not manually defined, but they will be trained
    • filters/neurons scans all image patches with same parameters
    • Convolutions followed by non-linear activation

    \[ Z = W \ast X + b \to ReLU(Z) \]

More Jargon: Padding & Pooling

  • padding
    • avoid size reduction
    • padding types: zero/constant, replicate, reflect, …
  • pooling filter (Pool)
    • spatial downsampling
    • channel depth stays the same
    • pooling type: max, average

Outputs Dimensions from Conv Layer

Output Size: \[ L_{out} = \frac{L_{in} - F + 2P}{S} + 1 \]

Note
  • \(L_{in}, L_{out}\) spatial dimensions of input and output images: \((H,W)\)
  • \(F=\) filter size, \(S=\) stride, \(P=\) padding
  • \((L_{in} - F + 2P)/S\) has to be integer for valid convolutional layer

Receptive Field

  • denotes the effective number of input pixels that contribute to activation
  • stacks of convolutional layers increase receptive field
  • early convolutional layers have small receptive fields (e.g \(3 \times 3\)): \(\to\) edge detection
  • deeper convolutional layers “see” more global structure \(\to\) object detection + classification accuracy

Habits & Recommendations

  • use input size \(L \times L = 2^n\) (e.g. 512)
  • use stride \(S=1\)
  • use padding \(P= (F-1)/2\) to retain input size \(\to\) multiple CONV layers
  • Typical structures: Input - Conv/Relu - Conv/Relu - Pool - Conv/Relu - … - Dense

A special one: 1x1 Convolutions

\(K \times K\) convolutions reduce spatial dimensions and increase channel depth (capactiy)

For very deep networks we may also want to reduce (summarize) channels (\(C \to K\)) without affecting spatial structures.

\[ y_{ij}^{(k)} = \sum_{c=1}^C w_c^{(k)} x_{ijc} + b_k \]

Note
  • efficient way to summarize feature
  • “feature map pooling”: pool only across channels
  • learnable weight \(w_c^{(k)}\) (in contrast to spatial pooling: Max/Average)

## 1D Conv

Convolutions combine two functions by **sliding** one (the kernel $g(x)$) over the other


**An image:**

![1D Convolution](images/1D_Conv.jpeg)

**A formula:**
$$
(f \ast g)(x) = \sum_{k=-\infty}^{\infty} f(k) \cdot g(x-k)
$$

sum is over all $k$, but in practice $g(x-k) = 0$ when $|x-k| \gg 0$

This can be used conveniently for

- smoothing/averaging
- change detection (derivatives) 
- pattern detection
- sharpening

### Generalization & New Jargon

![Convolution of RGB input with 2 filters. Filter depth matches input channels.](images/Conv_kernel.png)

- **input channels:** RGB images [C=3, H, W]

- **kernels=filters=neurons**:  [K,C,3,3]

  - filters have same channel depth as input
  - filter parameters are not manually defined, but they will be trained
  - filters/neurons scans all image patches with **same** parameters
  - Convolutions followed by non-linear activation

  $$
  Z = W \ast X + b \to ReLU(Z)
  $$
