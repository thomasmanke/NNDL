Goal

Tip

Want to make Gradient Descent faster and more robust

Use Mini-Batches

Calcuating the gradient for the full dataset can be expensive.

  • Break task and data into mini-batches: \(N_b < N\) (e.g. \(N_b = 64\))
  • gradient estimates will be more noisy

\[ \frac{dL}{dW} \approx g_t = \frac{1}{N_b}\sum_{i}^{N_b} \frac{dL_i}{dW} \]

Notice: \(dL/dW\) denotes gradient vector for all parameters (weights and biases)

Stochastic Gradient Descent (SGD)
  • more rapid
  • more memory efficient
  • escape shallow local minima
  • noisy updates as regularization
  • early convergence
  • efficiency more important than exactness !

Consider Momentum

Code 
import numpy as np
import matplotlib.pyplot as plt

w1 = np.linspace(-1, 1, 401)
w2 = np.linspace(-1, 1, 401)
W1, W2 = np.meshgrid(w1, w2)
sigma1 = 0.5   # wide in w1
sigma2 = 0.2   # narrow in w2
Z = 1 - np.exp(-((W1**2) / (2 * sigma1**2) + (W2**2) / (2 * sigma2**2)))

fig, ax = plt.subplots(figsize=(7, 5))
cont = plt.contourf(W1, W2, Z, cmap="Reds", levels=10)
ax.set_title("Loss Function (contour plot)")
ax.set_xlabel("w1")
ax.set_ylabel("w2")
plt.colorbar(cont, ax=ax, label="loss")
plt.show()
  • different parameters can have different impact on loss function
  • adjust learning rate or gradient depending on parameters (faster/slower learning)
  • prevent overshooting/oscillation in “narrow” directions
Exponential weighted averages

Average gradient estimates over minibatch iterations \(g_t \to v_t\)

\[ \begin{aligned} v_t &= \beta_1 v_{t-1} + (1-\beta_1) g_t \\ W &= W - \alpha v_t \end{aligned} \]

  • smoothed gradient (“velocity”)
  • new hyperparameter: e.g. \(\beta_1=0.9\)
  • weighted average over \(1/(1-\beta_1)\) batches
  • include friction term (\(\beta_1\)) and acceleration (\(1-\beta\))
Code 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)
x = np.linspace(0,64,65)
eps = np.random.normal(0, 0.2, size=x.shape)
# simulate decay to true value (y=1)
W = 1 + np.exp(-0.5*(x+eps))
G = np.random.normal(1, 0.2, size=x.shape)

beta = 0.9
G_smooth = np.zeros_like(G)
G_smooth[0] = W[0]
for t in range(1, len(G)):
    G_smooth[t] = beta * G_smooth[t-1] + (1 - beta) * G[t]

plt.scatter(x,G, label="mini-batch")
plt.plot(x, G_smooth, color='orange',label="smoothed")
plt.title(f'Exponential Weighted Average (beta = {beta})')
plt.xlabel('minibatch t')
plt.ylabel('gradient')
plt.legend()
plt.grid(True)
plt.show()
Note
  • efficient implementation of averaging
  • bias for small time \(t\) (early minibatches)
  • averages out strong fluctuations

RMSProp (root-mean-square prop)

Similar idea; scale gradients by their squared running average

\[ \begin{aligned} S_t &= \beta_2 S_{t-1} + (1-\beta_2) g_t^2 \\ W & = W - \alpha \left(\frac{g_t}{\sqrt{S_t}}\right) \end{aligned} \]

  • \(g_t^2\): element-wise squares of gradient
  • update direction of \(W\) with small gradients stronger than large gradients
  • dampen oscillations in high gradient directions
  • smoother updates permit increased learning rate

ADAM (Adaptive Moment Estimation)

\[ \begin{aligned} v_t &= \beta_1 v_{t-1} + (1-\beta_1) g_t \\ S_t &= \beta_2 S_{t-1} + (1-\beta_2) g_t^2 \\ W & = W - \alpha \left(\frac{v_t}{\sqrt{S_t}}\right) \end{aligned} \]

  • combines Momentum with RMSprop
  • real ADAM also contains some bias correction
  • most commonly used in modern application: (\(\beta_1 = 0.9, \beta_2=0.999)\), tune \(\alpha\)

Learning Rate Decay

Reduce learning steps closer to minimum

\[ \alpha_t = \alpha_0 \left(\frac{1}{1+\tau*{\mbox epoch}}\right) \]

  • Decay rate \(\tau\)
  • different ways to specify decay: exponential, manually

Local Minima & Plateaus

(Local) Ninima: - curvature (second derivative) has to be positive into all directions - increasingly unlikely in high-dimensions

Saddle Point:

  • curvature positive into some direction, negative in others
  • increasingly likely in high-dimensions
Code 
import numpy as np
import matplotlib.pyplot as plt

w1 = np.linspace(-3, 3, 400)
w2 = np.linspace(-3, 3, 400)
W1, W2 = np.meshgrid(w1, w2)

Z = W1**2 - W2**2

fig, ax = plt.subplots(figsize=(6, 5))
cont = ax.contourf(W1, W2, Z, levels=30, cmap="RdBu_r") 
plt.colorbar(cont, ax=ax, label="f(w1, w2)")

ax.contour(W1, W2, Z, levels=[0], colors='black', linewidths=1.2)
ax.axhline(0, color='k', lw=0.8)
ax.axvline(0, color='k', lw=0.8)

ax.set_aspect('equal', 'box')
ax.set_xlabel(r"$w_1$")
ax.set_ylabel(r"$w_2$")
ax.set_title("Saddle Point:  $Loss = w_1^2 - w_2^2$")
plt.show()
Note
  • In high dimensions of parameter space it is more common to have saddle points rather than local minima
  • Main concern is long plateaus = slow learning

Parameter Initialization

  • different initialization can lead to different solutions
  • avoid node symmetry: random initialization not zero
  • avoid vanishing gradients: no learning
  • avoid exploding gradients: unstable updates, divergent loss
  • chose activation function
Different Initializations at different layers:

For layer with \(n_l\) nodes \[ \begin{aligned} z &= \sum_i w_i x_i \\ Var(z) &= n Var(w_i) \end{aligned} \]

  • Xavier Initialization (for tanh): \(Var(w) = 1/n_l\)
  • Kaiming Initialization (for ReLU); \(Var(w) = 2/n_l\)
  • many more: e.g. torch.nn.init

Vectorization & GPU

Matrix Multiplication:

  • \(n\) samples
  • \(k\) input features
  • \(m\) output features

\[ \begin{aligned} z &= X &\cdot& W &+ b\\ [n,m] &= [n,k] &\cdot& [k,m] &+ [m] \end{aligned} \]

Be aware of conventions and possible matrix transposition.

Using GPU:

Code 
import torch
import torch.nn as nn

#detect GPU
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

model = nn.Linear(10, 2).to(device) # model --> Device
x = torch.randn(16, 10).to(device)  # data --> device
y = model(x)                        # output on device
print("y.device:", y.device)        # inspect device
y_cpu = y.cpu()                     # change device

## RMSProp (root-mean-square prop)

Similar idea; scale gradients by their squared running average

$$
\begin{aligned}
S_t &= \beta_2 S_{t-1} + (1-\beta_2) g_t^2 \\
W & = W - \alpha \left(\frac{g_t}{\sqrt{S_t}}\right)
\end{aligned}
$$

- $g_t^2$: element-wise squares of gradient
- update direction of $W$ with small gradients stronger than large gradients
- dampen oscillations in high gradient directions
- smoother updates permit increased learning rate


## ADAM (Adaptive Moment Estimation)

$$
\begin{aligned}
v_t &= \beta_1 v_{t-1} + (1-\beta_1) g_t \\
S_t &= \beta_2 S_{t-1} + (1-\beta_2) g_t^2 \\
W & = W - \alpha \left(\frac{v_t}{\sqrt{S_t}}\right)
\end{aligned}
$$

- combines Momentum with RMSprop
- real ADAM also contains some bias correction 
- most commonly used in modern application: ($\beta_1 = 0.9, \beta_2=0.999)$, tune $\alpha$



## Learning Rate Decay

Reduce learning steps closer to minimum

$$
\alpha_t =  \alpha_0 \left(\frac{1}{1+\tau*{\mbox epoch}}\right)
$$

- Decay rate $\tau$
- different ways to specify decay: exponential, manually

## Vectorization & GPU


**Matrix Multiplication**:

- $n$ samples
- $k$ input features
- $m$ output features

$$
\begin{aligned}
z &= X &\cdot& W &+ b\\
[n,m] &= [n,k] &\cdot& [k,m] &+ [m]
\end{aligned}
$$

Be aware of conventions and possible matrix transposition.
