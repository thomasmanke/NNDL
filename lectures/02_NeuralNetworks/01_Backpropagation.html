Goal

Find the gradients with respect to parameters. How much should paramters be changed to reduce the loss?

A simple network

The network encodes a calculation that is applied to each sample

Calculating Forward: Computational Graph

The loss function: \[ L(a,y) = -y \log(a) - (1-y)\log(1-a) \]

Calculating Backward

The derivatives can be calculates \[ \begin{aligned} \frac{\partial L}{\partial a} &= -\frac{y}{a} + \frac{1-y}{1-a}\\ \\ \frac{\partial L}{\partial z} &= \frac{\partial L}{\partial a}\frac{\partial a}{\partial z} = \frac{\partial L}{\partial a}a(1-a)\\ \\ \frac{\partial L}{\partial w_1} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial w_1} = \frac{\partial L}{\partial z} x_1\\\\ \frac{\partial L}{\partial w_2} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial w_2} = \frac{\partial L}{\partial z} x_2\\\\ \frac{\partial L}{\partial b} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial b} = \frac{\partial L}{\partial z} 1 \end{aligned} \]

Backward Calculation
  • derivatives = lookup and multiplication
  • decompose larger calculation into smaller local parts
  • strict computational flow ensures that all required contributions are known
  • importance of differentiable activation functions: \(\frac{\partial a}{\partial z}\)
  • attaches a gradient to each forward value: \(a \to da \equiv \frac{\partial L}{\partial a}\)
  • use matrices for book-keeping
  • flexibility: other loss functions, other inputs, more layers
  • this is for one sample \((\mathbf{x},y)\) only

Summing over all sample

\[ L = \frac{1}{N} \sum_i L(\hat y_i, y_i) \]

Note
  • for iid samples: maximizing log-likelihood
  • gradients accummulate: e.g. \(\frac{\partial L}{\partial b} = \sum_i \frac{\partial L_i}{\partial b}\)
  • in practice this is done for **batches* of training samples

Summary

Note
  • gradients are byproduct of backpropagation
  • shapes of gradients agree with shape of tensors
  • matrix notation for simplification and vectorized speedup
  • lookup and matrix multiplication

observed data: \(x\) true label: \(y\) predicted label probability: \(\hat y = f(x, \theta)\) \[ p(y|x) = \hat{y}^y (1-\hat{y})^y \to \mbox{maximize !} \]

Loss for one sample \(x\) \[ L(\hat y, y) = - \log p(y|x) = - y \log \hat y - (1-y) \log (1 - \hat y) \to \mbox{minimize !} \]

Loss for all \(N\) samples \((x_1,y_1), (x_2,y_2) \ldots (x_N, y_N)\)

\[ L(, x) = \frac{1}{N} \sum_i L(\hat y_i, y_i) \]

Overview

  • from a single neuron to network
  • from numbers to matrices
  • stacked layers of computation (passing information)
  • NN = matrix multiplications
  • NN = complex, non-linear functions

GPU and vectorized implementation

  • matrix multiplication for one sample x = column vector [m x 1]
  • matrix multiplication for n samples x = [m x n] matrix

Initialistion

  • different initialization can lead to different (local) optima in non-convex landscapes

  • avoid vanishing/exploding gradients

  • Xavier Initialization (for tanh): \(Var(w) = 1/n\)

  • Kaiming Initialization (for ReLU); \(Var(w) = 2/n\)

Activations Functions

Aim: introduce non-linearity

\[ \begin{aligned} z_1 & = W_1 \cdot x + b_1 \\ z_2 & = W_2 \cdot z_1 + b_2 \\ & = W_2 ( W_1 \cdot x + b_1) + b_2 \\ &= W^\prime x + b^\prime \end{aligned} \]

Use (non-linear) activation functions

  • only exception: final layers for regression problems

## Calculating Backward

The derivatives can be calculates
$$
\begin{aligned}
\frac{\partial L}{\partial a} &= -\frac{y}{a} + \frac{1-y}{1-a}\\ \\
\frac{\partial L}{\partial z} &= \frac{\partial L}{\partial a}\frac{\partial a}{\partial z} = \frac{\partial L}{\partial a}a(1-a)\\ \\
\frac{\partial L}{\partial w_1} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial w_1} = \frac{\partial L}{\partial z} x_1\\\\
\frac{\partial L}{\partial w_2} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial w_2} = \frac{\partial L}{\partial z} x_2\\\\
\frac{\partial L}{\partial b} &= \frac{\partial L}{\partial z} \frac{\partial z}{\partial b} = \frac{\partial L}{\partial z} 1 
\end{aligned}
$$

### Backward Calculation

- strict computational flow ensures that all required contributions are known
- importance of **differentiable** activation functions: $\frac{\partial a}{\partial z}$

- use matrices for book-keeping
- flexibility: other loss functions, other inputs, more layers
- this is for one sample $(\mathbf{x},y)$ only
:::

\begin{aligned}
z_1 & = W_1 \cdot x + b_1 \\
